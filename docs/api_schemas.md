# API Schemas

## 1. Objective

The API Schemas document defines the standardized request and response structures used by the AI-powered Applicant Tracking System (ATS). These schemas ensure consistent communication between system components, simplify data validation, and support seamless integration with external applications.

---

## 2. Module Overview

The ATS consists of multiple APIs responsible for resume upload, resume parsing, ATS scoring, and candidate ranking. Each API exchanges data using predefined JSON schemas to maintain consistency, improve reliability, and reduce integration errors.

---

## 4. Available API Schemas

| Schema | Purpose |
|---------|---------|
| upload_request_schema.json | Resume upload request |
| upload_response_schema.json | Upload confirmation response |
| parsing_response_schema.json | Structured resume parsing output |
| ats_score_schema.json | ATS score generation |
| ranking_schema.json | Candidate ranking results |

---

## 5. Schema Descriptions

### Resume Upload Request

Stores the information required to upload a resume into the ATS.

Example fields:

- Candidate Name
- Resume File
- File Type
- Upload Timestamp

---

### Upload Response

Returned after a successful upload.

Example fields:

- Upload Status
- Message
- Resume ID

---

### Resume Parsing Response

Represents the structured information extracted from the uploaded resume.

Example fields:

- Personal Information
- Skills
- Experience
- Education
- Certifications
- Projects

---

### ATS Score Schema

Stores the calculated ATS evaluation.

Example fields:

- Skill Score
- Experience Score
- Education Score
- Semantic Similarity
- Final ATS Score

---

### Candidate Ranking Schema

Represents the final ranking generated by the ATS.

Example fields:

- Candidate Name
- ATS Score
- Rank
- Recruitment Status

---

## 6. Validation Rules

The schemas ensure that:

- Required fields are present.
- Data types are validated.
- Missing values are handled safely.
- Invalid requests are rejected.
- API responses remain consistent across all modules.
